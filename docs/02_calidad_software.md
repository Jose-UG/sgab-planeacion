# Calidad de Software

La calidad del sistema no se limita a que funcione correctamente, sino que cumpla las características que garanticen el uso efectivo y la sostenibilidad en el tiempo.

Podemos definir tres atributos críticos para el SGAB.

---

## 1. Seguridad

El sistema gestionará información académica sensible, tales como los datos personales y calificaciones. Por esto mismo, es indispensable implementar mecanismos de autenticación y control de acceso basado en roles.

Si la seguridad no se garantiza, puede ocurrir:

- Accesos no autorizados.
- Manipulación de calificaciones.
- Pérdida de confianza institucional.

La seguridad en un sistema puede impactar la integridad y confidenciabilidad de la información.

---

## 2. Usabilidad

El sistema será utilizado por usuarios con distintos niveles de experiencia tecnológica. La interfáz debe ser clara, intuitiva y consistente.

La mala usabilidad podría generar:

- Errores en captura de datos.
- Resistencia al uso del sistema.
- Incremento en soporte ténico.

La facilidad de uso de un sistema es clave para su adopción y éxito a largo plazo.

---

## 3. Confiabilidad

El sistema debe ser confiable, es decir, debe funcionar correctamente bajo condiciones normales de uso y manejar adecuadamente los errores.

La falta de confiabilidad puede resultar en:

- Pérdida de datos.
- Interrupciones en el servicio.
- Frustración de los usuarios.

Un sistema confiable es esencial para mantener la confianza de los usuarios y garantizar la continuidad del servicio.

---

Estos atributos son prioritarios en esta fase inicial del proyecto, ya que sentarán las bases para el desarrollo y la implementación de un sistema robusto y efectivo.
